# AGENTS.md — workspace-lite

## Project

Google Workspace MCP proxy servers. 8 services (Drive, Gmail, Calendar, Sheets, Slides, Docs, Tasks, Forms) each with an Apps Script web app + Node.js MCP server package.

## Prerequisites

- Node.js >= 20
- npm
- clasp (`npm install -g @google/clasp`)
- Python 3 + venv (for mkdocs docs only)

## Common Tasks

### Build and validate
```bash
npm run build && npm run validate
```

### Deploy code changes to Apps Script
```bash
# Use the hardened helper scripts:
skills/workspace-lite-installer/scripts/deploy-all.sh . "Brief change description"
skills/workspace-lite-installer/scripts/deploy-single.sh . <service> "Brief description"
skills/workspace-lite-installer/scripts/verify-deployments.sh .
```

### Quick check
```bash
npm run typecheck && npm test
```

## Architecture

- `shared/src/` — TypeScript shared library (proxy client, schemas, response formatting, tool helpers)
- `shared/apps-script/` — Google Apps Script shared templates (Auth.gs, Response.gs)
- `packages/<service>/src/` — MCP server TypeScript source (tool registration)
- `packages/<service>/apps-script/` — Apps Script web app (.gs files)
- `packages/<service>/dist/` — Compiled MCP server output (gitignored)
- `config/service-registry.json` — Canonical service metadata
- `skills/` — OpenCode agent skills

### Flow
1. OpenCode MCP client → calls tool like `google-drive:drive_create_file`
2. Node.js MCP server → sends JSON POST to Apps Script web app URL (from .env)
3. Apps Script `doPost` → `Auth.gs` validates token class → `Policy.gs` checks action policy → `*Service.gs` executes
4. Response flows back through JSON

## Token Classes

`DEFAULT_AUTH_TOKEN_CLASSES` in `packages/<service>/apps-script/Auth.gs` controls primary token permissions:
- `read,draft` — read-only
- `read,draft,write` — read + write
- `read,draft,write,destructive` — above + trash/delete
- `read,draft,write,destructive,share` — above + sharing
- `read,draft,write,destructive,share,send` — all operations

## Important Rules

- Never commit `.env`, `.clasp.json`, `.clasprc.json`, or `BootstrapSecret.gs`
- `BootstrapSecret.gs` is gitignored but generated by `scripts/setup.sh` — never stage it
- Policy.gs, Response.gs, and Auth.gs must stay byte-identical across all 8 services
- Always run `verify-deployments.sh` after deploying
- Do not use `@HEAD` deployment — use versioned deployment IDs from `.env`
